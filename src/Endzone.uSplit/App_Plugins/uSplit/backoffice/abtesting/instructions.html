uSplit - A/B Testing for Umbraco

Using uSplit

Pre-requisites

uSplit integrates with Google Analytics Experiments to provide your site with A/B testing. Your site needs to have Google Analytics installed and a behavioural experiment needs to be created for uSplit to function.

Site setup

To enable uSplit you will need to include the macro provided by this plugin in your sites master template and create a property on your document types where the experiment code will be entered (and thus A/B testing enabled; no key = no test).

  1. Add the "A/B Redirect Experiment" macro provided by this plugin into your master template. Place this macro as the first element in the <head> tag.
  2. When prompted for a an "Experiment key field alias", write down the alias of the document type property you will create in the next step, that will contain the actual experiment code. This is custom but "experimentKey" will do the trick.
  3. Edit your document type and add a new property of the "Google Experiment Key" data type. Give it the same alias as you provided as a parameter to the macro, e.g. "experimentKey".

Your site is now ready to perform A/B tests. The macro will not execute if an experiment key is not found on the document, so it is safe to have it included in the master template, no coditional logic is required to disable the macro. To enable an A/B test for a particular page, you simply need to provide a valid experiment key to the property created in the last step.

Experiment setup

  1. Umbraco
    1. Identify the page you want to A/B test and create a number of variants of this page by copying it and publishing the variants under unique URLs.
    2. Take note of the original page URL and of the URLs of the variants. (e.g. www.mycoolsite.com/home, www.mycoolsite.com/home-v1, www.mycoolsite.com/home-v2)
  2. Google Analytics
    1. Log into Google Analytics and navigate to your site.
    2. Open the "Reporting" section of Google Analytics.
    3. Under "Behaviour" select "Experiments" and create a new experiment.
    4. Follow this detailed guide on how to set up your experiment
    5. On "Step 2" you will need to provide URLs for the original page and for the variants.
    6. On "Step 3", take note of the Experiment Key provided. To display the key, you need to select the "Manually insert the code" installation option. The key is in the form of "123456789-0"
    7. Go back to your Umbraco site to paste in the experiment key to the original page (see bellow).
  3. Umbraco
    1. Edit the original page (and only this one, not the variants), and insert the Experiment Key into the property created during site setup.
    2. Publish the page. The macro will start including the Google Analytics Experiment javascript as part of the page now.
    3. Let's validate and start the experiment on Google Analytics.
  4. Google Analytics
    1. Proceed to "Step 4", save the experiment review the status of the experiment. You should see ticks next to your original page (Google Analytics and experiment code detected) as well as the variants (Google Analytics detected).
    2. Start the experiment.
    3. Profit.
